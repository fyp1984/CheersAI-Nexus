## 1. 项目概述

### 1.1 项目背景

CheersAI 产品矩阵持续扩展，目前包括：

| 产品                   | 定位              | 状态  |
| -------------------- | --------------- | --- |
| **CheersAI Desktop** | 一站式 Agent 办公客户端 | 已上线 |
| **CheersAI Vault**   | 企业级数据安全存储       | 规划中 |
| **CheersAI Writer**  | AI 写作助手         | 规划中 |
| **CheersAI SSO**     | 统一身份认证          | 开发中 |

随着产品矩阵扩展，迫切需要建立**统一运营管理平台**，解决以下问题：

* ❌ 各产品用户体系独立，无法统一管理

* ❌ 会员权益分散，无法跨产品共享

* ❌ 运营数据分散，缺乏全局视角

* ❌ 用户反馈分散，响应效率低

* ❌ 审计日志分散，合规风险高

### 1.2 项目目标

**核心目标**: 构建 CheersAI 产品运营驾驶舱

### 1.3 项目范围

#### 包含（In Scope）

* ✅ 统一账号系统（注册、登录、SSO）

* ✅ 产品管理（产品配置、版本管理）

* ✅ 用户管理（用户列表、状态管理）

* ✅ 会员体系（等级、权限、额度）

* ✅ 功能定价（订阅、按量、套餐）

* ✅ 用户反馈（收集、处理、闭环）

* ✅ 审计日志（行为日志、系统日志）

* ✅ 公告系统（系统通知、产品更新）

* ✅ 数据分析（DAU/MAU、付费率、留存）

#### 不包含（Out of Scope）

* ❌ 各产品的业务功能开发

* ❌ 移动端 App 开发

* ❌ 第三方支付系统开发（使用现有方案）

* ❌ 客服系统（使用第三方工具）

### 1.4 产品命名

**产品名称**: CheersAI Nexus

**命名含义**:

* **Nexus** = 中枢 / 枢纽 / 连接点

* 表达 CheersAI 产品生态的**核心控制中心**

* 所有产品通过 Nexus 连接，实现统一管理

**域名规划**:

* 运营后台: `nexus.cheersai.com`

* 统一认证: `auth.cheersai.com`

* API 网关: `api.cheersai.com`

## 2. 产品定位

### 2.1 产品愿景

**"一个平台，掌控全局"** - 为 CheersAI 产品矩阵提供统一的运营管理能力

### 2.2 产品定位

| 维度       | 定位                 |
| -------- | ------------------ |
| **产品类型** | B 端运营管理平台（内部使用）    |
| **核心价值** | 统一账号 + 统一会员 + 统一运营 |
| **技术架构** | 微服务 + SSO + 共享数据库  |
| **部署方式** | 私有化部署（自有服务器）       |

### 2.3 产品定位对比

| 维度       | 无 Nexus（当前） | 有 Nexus（目标） |
| -------- | ----------- | ----------- |
| **账号体系** | 各产品独立注册     | 一次注册，全产品通用  |
| **会员权益** | 各产品独立付费     | 一次付费，权益共享   |
| **运营数据** | 分散在各产品      | 统一驾驶舱，全局视角  |
| **用户反馈** | 各产品独立处理     | 统一收集，高效响应   |
| **审计合规** | 日志分散，难以追溯   | 统一审计，合规保障   |
| **运营效率** | 低（需登录多个后台）  | 高（一个平台搞定）   |

### 2.4 与其他产品的关系

**关系说明**:

* Nexus 是**管理平台**，不直接面向终端用户

* Desktop/Vault/Writer 是**业务产品**，面向终端用户

* 所有产品通过 **SSO** 共享用户认证

* 所有产品通过 **API** 获取会员权限

| expire\_at       | Timestamp | 否  | 过期时间                               |
| created\_by      | UUID      | 是  | 创建人                                |
| created\_at      | Timestamp | 是  | 创建时间                               |

### 6.9 模块 8: 数据分析

#### 功能描述

提供运营数据分析和可视化报表。

#### 核心指标

| 指标       | 说明     | 计算方式           |
| -------- | ------ | -------------- |
| **DAU**  | 日活跃用户  | 当日登录的独立用户数     |
| **WAU**  | 周活跃用户  | 近 7 天登录的独立用户数  |
| **MAU**  | 月活跃用户  | 近 30 天登录的独立用户数 |
| **新增用户** | 新注册用户数 | 当日/周/月新注册用户    |
| **付费用户** | 付费会员数  | 当前有效付费会员数      |
| **付费率**  | 付费转化率  | 付费用户 / 总用户     |
| **ARPU** | 平均用户收入 | 总收入 / 活跃用户数    |
| **留存率**  | 用户留存   | 次日/7日/30日留存    |

#### 数据看板

| 看板       | 内容             | 更新频率 |
| -------- | -------------- | ---- |
| **总览看板** | DAU/MAU、新增、付费率 | 实时   |
| **用户看板** | 用户增长、留存、分布     | 每小时  |
| **收入看板** | 收入趋势、ARPU、付费分布 | 每日   |
| **产品看板** | 各产品使用量、排行      | 每小时  |

### 10.2 安全策略

#### 认证安全

| 策略       | 配置             | 说明    |
| -------- | -------------- | ----- |
| 密码强度     | 最少 8 位，含大小写+数字 | 强制要求  |
| 登录失败锁定   | 5 次失败锁定 30 分钟  | 防暴力破解 |
| 会话超时     | 2 小时无操作自动登出    | 安全保护  |
| 多设备登录    | 允许，但限制 3 个设备   | 可配置   |
| 敏感操作二次验证 | 修改密码、删除数据      | 强制要求  |

#### 数据安全

| 策略   | 说明                |
| ---- | ----------------- |
| 传输加密 | 全站 HTTPS，TLS 1.3  |
| 存储加密 | 敏感数据 AES-256 加密   |
| 密码存储 | bcrypt 哈希，cost=12 |
| 日志脱敏 | 敏感字段自动脱敏          |
| 数据备份 | 每日全量 + 实时增量       |
